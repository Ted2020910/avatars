# Document 节点 — 原始事实与资源记录

Document 节点只陈述事实，不做推导。内容来自外部资料、调研结果、既有约束等。

写作原则：
- 只写"是什么"，不写"为什么"或"怎么做"
- 来源明确时注明来源
- 内容保持原始性，推导分析放到 Explore 节点

典型用途：
- 记录用户访谈原始内容
- 存放技术文档摘录
- 保存市场数据或竞品信息
- 记录项目约束条件（时间、预算、技术栈）
