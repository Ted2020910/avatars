# AgentHarness hook 系统设计：context / tool_call / tool_result 三个 hook 各承载什么逻辑

## 真实代码中的 hook 调用位置

`agent-harness.ts` 构造函数里硬连接了三个 hook 到 Agent 的三个回调（第 204-230 行）：

```ts
// hook 1：transformContext → context hook
this.agent.transformContext = async (messages) => {
    const result = await this.emitHook({ type: "context", messages: [...messages] });
    return result?.messages ?? messages;
};

// hook 2：beforeToolCall → tool_call hook（可 block）
this.agent.beforeToolCall = async ({ toolCall, args }) => {
    const result = await this.emitHook({
        type: "tool_call",
        toolCallId: toolCall.id,
        toolName: toolCall.name,
        input: args as Record<string, unknown>,
    });
    return result ? { block: result.block, reason: result.reason } : undefined;
};

// hook 3：afterToolCall → tool_result hook（可改写结果）
this.agent.afterToolCall = async ({ toolCall, args, result, isError }) => {
    const patch = await this.emitHook({
        type: "tool_result",
        ...
    });
    return patch ? { content: patch.content, details: patch.details, ... } : undefined;
};
```

注册方式（外部使用）：
```ts
harness.on("tool_call", async ({ toolName, input }) => {
    if (isDangerousCommand(input)) return { block: true, reason: "权限不足" };
});

harness.on("tool_result", async ({ toolName, content }) => {
    if (content.length > MAX_CONTENT) return { content: truncate(content) };
});
```

## 需要决策的问题

### context hook 承载什么？

候选逻辑：
- **token 估算裁剪**：如果 messages token 总量超过阈值，从最早的对话开始删除
- **动态系统提示注入**：往 messages 里插入当前时间、用户信息等上下文
- **compaction 触发判断**：检测到超限时标记需要 compact，但不在这里执行

### tool_call hook 承载什么？

候选逻辑：
- **权限检查**：某些工具只能在特定角色/模式下执行
- **危险操作确认**：破坏性操作（删文件、运行命令）需要 block 并请求用户确认
- **日志记录**：记录工具调用到审计日志

### tool_result hook 承载什么？

候选逻辑：
- **结果截断**：shell 输出超过 N 行时截断，避免撑爆 context window
- **内容脱敏**：去掉 secret、token 等敏感信息
- **terminate 判断**：检测到特定结果时（如任务完成标记），设置 terminate=true
