# AgentTool 接口设计：execute + details 泛型如何为 UI 和日志服务

## 真实代码中的接口定义

`types.ts` 第 353-376 行：
```ts
interface AgentTool<TParameters extends TSchema = TSchema, TDetails = any> extends Tool<TParameters> {
    label: string;                    // UI 展示用（不发给 LLM）
    prepareArguments?: (args: unknown) => Static<TParameters>;  // 参数 shim
    execute: (
        toolCallId: string,
        params: Static<TParameters>,
        signal?: AbortSignal,
        onUpdate?: AgentToolUpdateCallback<TDetails>,  // 流式进度回调
    ) => Promise<AgentToolResult<TDetails>>;
    executionMode?: "sequential" | "parallel";
}

interface AgentToolResult<T> {
    content: (TextContent | ImageContent)[];  // 发给 LLM 的文本/图片
    details: T;                               // 结构化附加数据，给 UI 和日志用
    terminate?: boolean;                      // 是否提前终止 agent loop
}
```

**关键区分**：
- `content` → 发给 LLM，必须是文本/图片
- `details` → 不发给 LLM，给 UI 渲染和日志记录用

## details 字段在代码里的流转

1. `tool.execute()` 返回 `AgentToolResult<TDetails>`（含 details）
2. `agent-loop.ts` 的 `afterToolCall` 钩子可以修改 details（第 667 行）
3. `createToolResultMessage()` 把 details 放进 `ToolResultMessage.details`（第 709 行）
4. `processEvents("message_end")` 时 `Agent._state.messages` 会保存这条含 details 的 ToolResultMessage
5. UI 层通过订阅 `tool_execution_end` 事件拿到 `result.details`

### onUpdate（流式进度）什么情况下需要？

- 长时间运行的工具（shell 执行、网络请求）需要 onUpdate 向 UI 推送中间状态
- 短时工具不需要
