# avatars
现有的openclaw类的功能目前还有一些限制，以下是我觉的还不足的一些点：
1. 代码不够解耦，不是面向ai编程的设计
2. 没有分身的功能，上下文太长，过于消耗tokens
3. 没有组织的概念
4. 没有技能的自动进化
5. 反馈太少，很多时候不知道模型在不在干活
6. 记忆机制还是过于简洁，很多时候也不知道她有没有记住，需要反复调教
7. 用户对于消耗量的感知不清晰，很多时候不知道消耗了多少钱，不知道何时该切换模型，切换模型也应该是一个技能
8. 不清楚模型的上下文是如何加载的，如何加载soul.md,有哪些skills，哪些skills被调用了

因此希望可以有这样的使用体验
## 希望可以有的交互体验
1. 可以实现每一个agent的创建，任务的追踪，组织的添加，不同身份角色的任务分配，不的技能自动进化，根据任务的完成情况，自动添加新的技能
3. 可以将skill归属到不同用户，然后使用git的方式进行管理，实现技能的版本控制，同时也可以实现技能的分享和协作
4. 可以实现类似于火影的分身技能
5. agent可以自己打造工具，给自己使用，可以有一个类似于锻造师的角色
6. 组织可以称为宗门
7. 可以有不同的仙元能量，不同的模型使用不同的仙元能量，好的模型仙元能量的质量更高，但是成本更快
8. agent需要可以自己更新记忆，更新学习技能
9. agent需要有修炼的环节，可以更新自己的记忆，或者更新自己的技能。
10. skills需要有实验的重试，检查skill效果（使用git管理）
### web-ui
1. 用户可以在web-ui上追踪每个agent的上下文构建过程，可以看到这个agent有哪些技能，哪些技能正在被调用
2. 用户可以在web-ui上手动切换agent的仙元能量，也可以让agent自己切换仙元能量（这个可以自己实现，根据模型的性能，自动切换仙元能量）
3. 可以追踪不同的项目有哪些智能体在工作，有哪些组织，有哪些任务在进行，有哪些任务已经完成
4. 用户可以在web-ui上手动添加任务，也可以让agent自己添加任务
5. 用户也可以查看有哪些组织，有哪些智能体，有哪些任务
6. 需要有分身的概念，分身需要涉及到，分身的创建，分身的切换，运行，合并
